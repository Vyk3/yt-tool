# Fast-Path Command Protocol

本文件定义低延迟命令执行的规范层，供 Claude / Codex 运行时适配。它描述“哪些命令适合走快捷通道”以及“如何安全退化”，不假设某个宿主已经支持 `!` 前缀。

## 目标

- 减少高频、低风险命令的额外解释成本
- 缩短查状态、跑固定验证、搜信息的往返延迟
- 保持输出可审计、可回放、可控参数范围

减少模型开销不是唯一目标。任何快捷执行都不能牺牲边界控制和结果可追溯性。

## 适合走快捷通道的命令类别

### 状态查询

- `git status --short`
- `git branch --show-current`
- `git diff --stat`

### 固定验证

- `ruff check app/ tests/`
- `python -m pytest tests/ -q`
- `bash tools/check_ci.sh`

### 信息检索

- `rg`
- `rg --files`

默认要求：命令本身已在仓库或团队约定中稳定存在，不依赖临场拼接复杂参数。

## 协议要求

### 允许的参数模式

- 仅允许固定模板或收缩后的有限参数集
- 优先使用仓库脚本入口，而不是直接暴露任意 Bash
- 需要变参时，参数空间必须可枚举或可显式验证

### 输出上限

快捷通道必须有输出预算，避免把完整日志直接灌回主会话。

建议策略：

- 默认返回摘要 + 关键信号
- 超出上限时自动截断，并注明已截断
- 原始命令和截断前的统计信息应可审计

### 自动摘要

按命令类型返回最小必要结果：

- 状态查询：当前状态 + 变化数量 + 阻断项
- 测试 / 检查：通过 / 失败、失败点、关键命令
- 搜索：匹配文件、关键定位、匹配数量

禁止把长日志原样当作“摘要”返回。

### 失败回退

出现以下任一情况时，必须回退到常规工具调用或显式人工确认：

- 参数超出固定模板
- 输出超过摘要策略可控范围
- 命令涉及写操作或远程操作
- 结果需要保留完整上下文才能判断
- 当前运行时不支持该快捷协议

### 审计要求

每次快捷执行至少记录：

- 原命令
- 命令类别
- 摘要结果
- 是否发生截断
- 是否发生回退

如果摘要中包含推断，需明确标记为推断，不得伪装成原始输出。

## 与 allow 规则的关系

- 进入快捷通道的命令，仍需先满足 `rules/allowlist.md` 的准入标准
- 快捷通道只是一种执行路径，不自动放宽权限边界
- A / B 类命令优先纳入快捷协议；C / D 类默认不纳入

## 跨运行时适配建议

### 宿主支持快捷前缀时

- 将前缀映射到同一组命令分类
- 共享相同的输出预算、摘要规则和回退条件
- 不为不同宿主定义相互冲突的安全边界

### 宿主不支持快捷前缀时

- 退化为固定脚本入口或固定命令模板
- 仍沿用同一套分类、摘要和审计要求

## 首批候选

以下命令适合作为首批候选进行评估：

- `git status --short`
- `git branch --show-current`
- `git diff --stat`
- `rg --files`
- `ruff check app/ tests/`
- `python -m pytest tests/ -q`
- `bash tools/check_ci.sh`

这些命令的共同特征是：高频、边界清晰、输出可压缩、失败副作用低。

## 落地方式

本轮已在仓库内提供固定模板入口，但仍不实现宿主级快捷前缀。

推荐的落地顺序：

1. 先固定候选命令清单
2. 为每条候选补参数边界、摘要格式、回退条件
3. 优先收敛到仓库脚本或固定命令模板
4. 宿主支持时，再把同一协议映射到快捷前缀或低延迟执行入口

当前仓库内的最小入口为：

```bash
bash scripts/fast_path.sh status
bash scripts/fast_path.sh branch
bash scripts/fast_path.sh diffstat
bash scripts/fast_path.sh files
bash scripts/fast_path.sh files app
bash scripts/fast_path.sh lint
bash scripts/fast_path.sh test
bash scripts/fast_path.sh ci
bash scripts/fast_path.sh ci --json
bash scripts/fast_path.sh ci-watch
bash scripts/fast_path.sh ci-watch --interval 30 --timeout 180
```

该入口的约束是：

- 只接受固定子命令
- 只允许少量显式校验过的可选参数
- 不接受任意 shell 片段
- 超出模板能力时必须回退到常规命令调用

当前已显式允许的变参：

- `files [app|tests|scripts|rules|skills|agents]`
- `ci [--json]`
- `ci-watch [--interval N] [--timeout N]`

不在以上列表内的参数，默认拒绝。

## 本轮是否实现

默认结论：**不在本轮实现宿主级 Fast Path 执行**。

原因：

- 当前仓库只能稳定落规则、协议和固定入口，不能真正定义宿主执行层行为
- 如果现在直接实现一个“伪 Fast Path”，很容易退化成额外包装层，却没有真正减少交互成本
- 现阶段更值得先做的是把高频命令收敛为稳定入口，为后续宿主适配提供明确边界

## 本轮可实现的最小落地

本轮已完成的最小项：

- 为首批候选建立固定命令模板
- 为后续宿主适配保留统一命令分类

后续仍可继续补充：

- 对模板输出补更细的摘要预算说明
- 为少量允许变参的模板继续做显式参数校验

仍不建议在没有宿主支持的前提下，引入新的命令前缀语法或本地模拟层。
