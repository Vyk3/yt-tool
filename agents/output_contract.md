# Agent Output Contract

统一约束 `Scout`、`Reviewer` 及后续独立子 agent 的返回格式，目标是减少上下文噪音，同时保留可审计证据。

## 适用范围

- Claude Code 中的独立 Explore / Review / Rescue 流程
- Codex CLI 中的独立 subagent 返回主会话
- 主会话在向人类展示前的二次压缩

该协议不替代 `AGENTS.md` 的角色边界；它只约束“返回什么”和“怎么压缩”。

## 第一层：子 agent 返回主会话

子 agent 默认只返回结构化摘要，不回传完整推理过程。

最小字段：

- **结论**：一句话说明本次发现或判断
- **风险等级**：`critical` / `high` / `medium` / `low` / `info`
- **证据**：文件路径 + 行号，或命令名 + 关键结果
- **建议动作**：主会话接下来应做什么
- **可省略细节**：哪些分析过程可以不继续展开

推荐模板：

```text
结论：...
风险等级：...
证据：
- path:line
- command: summary
建议动作：...
可省略细节：...
```

### 强制约束

- 不默认返回完整 chain-of-thought 或大段分析过程
- 不粘贴整文件、整段 diff、长日志
- 不以“我看了很多地方但没有问题”替代证据摘要
- 结论必须能追溯到证据，证据必须能被主会话复核

### 例外情况

仅在以下情况允许附加少量必要细节：

- 高风险问题需要证明严重性
- 存在争议，需要给出最小对比片段
- 主会话明确要求展开某一证据链

即使在例外场景，也只附最小必要内容，不回传整段探索过程。

## 第二层：主会话展示给人类

主会话收到子 agent 摘要后，仍需二次压缩为“人可快速消费”的结果。

优先顺序：

1. 结论
2. 阻断项 / 风险
3. 下一步动作
4. 按需展开的证据

展示时应把探索细节降级，不抢占结论位置。

## 展示预算

### Scout / 探索摘要

- 默认 3-5 条关键结论
- 每条仅保留必要文件路径和行号
- 无需列出所有搜过的文件

### Reviewer / 审查结论

- 先列发现，再给简短总结
- 每条发现必须保留严重性、定位、为什么重要
- 不因压缩而省略未覆盖风险或缺失测试

### 验证结果

- 优先保留命令、结果、失败点
- 长日志只摘关键信号
- 成功场景只保留通过摘要，不回放完整输出

### 失败归因

- 先说根因，再说证据，再说止血和永久防护
- 时间线只保留影响判断的关键节点

## 角色补充要求

### Scout

- 只返回检索结论和证据定位
- 默认不回传完整文件内容
- 必须标注来源路径与行号

### Reviewer

- findings 优先，不得被概述吞没
- 即使压缩，也必须保留严重性、影响面、修复方向
- `PASS` 仅在无实质问题时使用

### Rescuer

- 诊断结论与替代方案分开表达
- 明确说明“已确认根因”还是“当前最强假设”

## 审计要求

- 主会话应保留子 agent 的摘要原文，避免二次转述失真
- 人类最终看到的版本可以压缩，但必须能回溯到子 agent 证据
- 如压缩过程中做了推断，应在展示时显式标为推断
